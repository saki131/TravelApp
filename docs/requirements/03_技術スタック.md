## 2. 外部 API 統合方針

### 2.1 Amadeus for Developers（フライト検索）

```
利用 API エンドポイント:
GET /v2/shopping/flight-offers          # フライト検索（同区間最安値）
GET /v1/shopping/flight-dates           # 価格カレンダー（日別最安値）
GET /v1/shopping/flight-inspirations    # F-02b: 日時固定→複数行先一覧
GET /v1/reference-data/locations        # 空港・都市サジェスト
```

- **認証**: OAuth 2.0 Client Credentials
- **レート制限**: 無料枠 2,000req/月 → **個人利用なら十分**（1日66回相当）
- **キャッシュ戦略**: 同一検索クエリは30分 DB キャッシュ

### 2.2 Trip.com Affiliate 連携

```
登録: Trip.com Partner Center (https://partners.trip.com/) に登録（無料）
取得できる情報:
  - フライト検索結果（アフィリエイトリンク付き）
  - セール / プロモーション情報
  - ホテル情報
特徴: アジア路線に強い（LCC 含む）。成約時にアフィリエイト報酬あり
```

### 2.3 Kiwi API（フォールバック）

```
Amadeus API → 失敗 or レート超過 → Kiwi (Tequila) API
```

### 2.4 Gemini Flash 連携

```python
from google import generativeai as genai

# AI旅行プランナー プロンプト例
# レスポンスは JSON 形式で受け取り、フライト検索結果と組み合わせる
```

- **無料枠**: Gemini 1.5 Flash — 1,500req/日・15RPM

### 2.3 セール情報収集（定期ジョブ）

```
APScheduler + Cloud Scheduler 構成:

  Cloud Scheduler（無料 3ジョブ）→ HTTP GET /internal/jobs/{job}
    - collect-sales  : 毎日 7時・12時・21時  （Gemini グラウンディング）
    - collect-rss    : 毎時              （JAL / ANA / Trip.com RSS）
    - cleanup        : 毎日 0時          （期限切れセール・キャッシュ削除）

  Cloud Run がコールドスタートで停止していても Cloud Scheduler からの
  HTTP リクエストでコンテナが起動 → APScheduler がジョブを即時実行する。
  （APScheduler のスケジュール管理は Cloud Scheduler に委譲する設計）
```

## 3. API 設計概要

### 3.1 ベース URL
```
/api/v1/
```

### 3.2 認証
```
個人利用のためフル認証は不要。
全エンドポイントに固定 API キー認証を適用:
  X-API-Key: <.env に記載した個人用 API キー>
```

### 3.3 主要エンドポイント

```
# フライト
GET  /flights/search          # 同区間のフライト検索
GET  /flights/price-calendar  # 価格カレンダー
GET  /flights/inspire         # F-02b: 日時固定→複数行先の最安値一覧
GET  /airports/suggest        # 空港オートコンプリート

# ホテル
GET  /hotels/search           # ホテル検索
GET  /hotels/{id}             # ホテル詳細

# セール
GET  /sales                   # セール一覧（カレンダー用）
GET  /sales/{id}              # セール詳細
GET  /sales/today             # 本日のセール

# AI
POST /ai/plan                 # Gemini による旅行プラン生成
POST /ai/suggest-destination  # 行先候補の AI 提案

# お気に入り
GET  /favorites               # お気に入り一覧
POST /favorites               # お気に入り追加
DELETE /favorites/{id}        # お気に入り削除

# 価格アラート
GET  /alerts                  # アラート一覧
POST /alerts                  # アラート作成
PUT  /alerts/{id}             # アラート更新
DELETE /alerts/{id}           # アラート削除
```

## 6. ローカル開発環境 (Docker Compose)

```yaml
# docker-compose.yml
services:
  backend:
    build: ./backend
    ports: ["8000:8000"]
    env_file: ./backend/.env
    # Redis・Celery Worker は不要（個人利用・APScheduler内蔵）

  frontend:
    build: ./frontend
    ports: ["3000:3000"]
    env_file: ./frontend/.env.local
    depends_on: [backend]
```

> Neon はクラウドホスティングのため docker-compose の DB コンテナは不要。
> `DATABASE_URL` に Neon 接続文字列を `.env` に設定するだけ。

## 7. Cloud Run デプロイ手順（KakeiboApp と同構成）

```powershell
cd "c:\Users\Owner\workspace\TravelApp\backend"

# 1. イメージビルド & push
gcloud builds submit --tag asia-northeast1-docker.pkg.dev/<GCPプロジェクト>/travelapp/backend:latest

# 2. Cloud Run にデプロイ
gcloud run deploy travelapp-backend `
  --image asia-northeast1-docker.pkg.dev/<GCPプロジェクト>/travelapp/backend:latest `
  --region asia-northeast1 `
  --set-env-vars "DATABASE_URL=...,AMADEUS_CLIENT_ID=..."
```

> GCP プロジェクトは KakeiboApp と共用または新規作成・要確認。
