# 旅行アプリ 要件定義書

**バージョン**: 1.1  
**作成日**: 2026年4月29日  
**最終更新**: 2026年4月30日  
**プロジェクト名**: TravelApp（仮称）

---

## 1. プロジェクト概要

### 1.1 目的
飛行機・ホテル・旅行パックのセール情報をカレンダー形式で一覧表示し、Skyscanner ライクなフライト検索機能を提供する旅行計画支援 Web アプリケーション。

### 1.2 ターゲットユーザー
- **個人利用（自分のみ）** ※マルチユーザー対応は不要
- 利用デバイス：**スマートフォン中心**（PWA 対応）
- できるだけ安く旅行したい。日程・行先を柔軟に変えて最安値を探すスタイル

### 1.3 解決する課題
| 課題 | 解決策 |
|------|--------|
| セール情報が各サイトに分散していて見つけにくい | カレンダーに集約して一覧表示 |
| 最安フライト日を探すのに手間がかかる | カレンダー上で価格を色分け表示 |
| 複数の予約サイトを比較するのが面倒 | 横断比較結果を一画面に表示 |

### 2.2 F-01: セール・クーポンカレンダー

#### 概要
航空券・ホテル・旅行パックのセール情報をカレンダー上に集約表示する。

#### 機能詳細

**カレンダー表示**
- 月表示・週表示・リスト表示の切り替え
- 各日付にセールイベントをバッジ表示
  - 飛行機セール（青）
  - ホテルセール（緑）
  - 旅行パック（オレンジ）
  - クーポン期限（赤）
- セールイベントをクリックすると詳細パネルを表示

**セール情報の詳細表示**
- セール名・割引率・割引額
- 期間（開始日〜終了日）
- 対象路線・目的地・ホテル
- 予約サイトへのリンク（外部遷移）
- クーポンコードのコピー機能

**フィルタリング**
- カテゴリ絞り込み（飛行機 / ホテル / 旅行パック）
- 出発地・目的地指定
- 割引率下限指定
- 価格上限指定

**データソース（外部連携）**

| 収集方法 | 対象 | 詳細 |
|----------|------|------|
| **RSS フィード** | JAL / ANA / Peach / Jetstar | セール**開催情報**（セール名・対象路線・予約期間・旅行期間・目安最低価格「¥X,XXX〜」）を定期取得。特定便のリアルタイム価格は含まれない |
| **Trip.com Affiliate API** | アジア LCC / ホテルセール | API 経由で正規取得 |
| **HIS / JTB アフィリエイト** | 旅行パック | アフィリエイト API 経由 |
| **Gemini Google Search グラウンディング** | **直販フラッシュセール** | 下記参照 |

> **RSS・グラウンディングで取得した価格はあくまで「目安の表示価格」であり、実際の予約価格は Amadeus API で取得する。**  
> カレンダー上のセールバッジ = セール開催の告知。実際の価格確認は F-02 / F-03（Amadeus API）で行う設計とする。

**Gemini Google Search グラウンディングによるフラッシュセール検知**

RSS や Affiliate API では取得できない**直販限定の格安フラッシュセール**（例: 「JAL SUPER VALUE 先得」「Peach タイムセール」など）を Gemini の Google Search グラウンディング機能でリアルタイム検索・検知する。

```
仕組み:
  1. APScheduler が毎日 7時・12時・21時に Gemini API を呼び出す
  2. Gemini が Google Search をリアルタイム検索:
     「JAL フラッシュセール 本日」「Peach タイムセール 開催中」等
  3. Gemini がセール情報（概要・期限・対象路線・URL）を JSON で返答
  4. DB に保存してカレンダーに表示

注意事項:
  - 価格の正確性は保証不可（公式サイトへのリンクで最終確認を促す）
  - Gemini 1.5 Flash 無料枠（1,500req/日）内で動作
  - グラウンディング機能は無料枠に含まれる
  - 取得できるのは「セール開催の検知・概要」であり価格検索は Amadeus API を使用
```

### 2.3 F-02: フライト検索

#### 概要
Skyscanner 相当のフライト検索機能。出発地・目的地・日程を指定して、複数の航空会社・予約サイトの価格を横断比較表示する。

#### 検索条件
| 項目 | 内容 |
|------|------|
| 旅行形態 | 片道 / 往復 / 多区間 |
| 出発地 | 空港コード・都市・日本語検索対応（例: 東京、TYO、HND） |
| 目的地 | 空港コード・都市（**「どこでも」選択で F-02b へ連携**） |
| 出発日 | 日付指定 / 月指定（最安値月を探す） |
| 帰着日 | 往復選択時のみ |
| 乗客数 | 大人のみ（個人利用につき簡略化） |
| キャビンクラス | エコノミー / ビジネス |
| 乗り継ぎ | 直行便のみ / 1回乗り継ぎ可 / 制限なし |
| 乗り継ぎ空港指定 | 乗り継ぎたい空港を指定可能 |

#### 検索結果表示
**一覧表示（リスト形式）**
- ソート：価格 / 所要時間 / 出発時刻 / 到着時刻 / 乗り継ぎ回数
- 各フライトカード表示内容：
  - 航空会社名とロゴ
  - 出発時刻・到着時刻
  - 出発空港・到着空港（IATA コード + 名称）
  - 所要時間
  - 乗り継ぎ回数・乗り継ぎ空港・乗り継ぎ待ち時間
  - 最安値と提供している予約サイト一覧
  - 「予約サイトへ」ボタン（外部遷移）

**価格カレンダービュー**
- ±15日間の日別最安値をカレンダー形式で表示
- 最安値の日を色（緑〜赤）で視覚表示

**フィルタパネル**
- 価格範囲スライダー
- 出発時刻帯（早朝 / 午前 / 午後 / 夜間）
- 到着時刻帯
- 直行便のみトグル
- 乗り継ぎ空港チェックボックス
- 航空会社チェックボックス
- 予約サイトチェックボックス

#### 航空会社カバレッジ

| API | カバー範囲 | 代表的な航空会社 |
|-----|-----------|----------------|
| **Amadeus**（メイン） | GDS 参加 900社以上 | JAL / ANA / ZIPAIR / United / Delta / British Airways / Lufthansa 等フルサービス航空会社全般 |
| **Kiwi**（フォールバック） | GDS 非参加 LCC を含む | Peach / Jetstar / AirAsia / Scoot / Cebu Pacific 等アジア LCC |
| **Trip.com Affiliate** | アジア特化 | 春秋航空 / 吉祥航空 / アジア格安 LCC |

> **API で取得できないケース**（直販フラッシュセールのみの便など）は、F-01 の Gemini グラウンディングでセール開催を検知し、公式サイトリンクへ誘導する。また検索結果下部に「他のサイトでも確認する」として Trip.com / Skyscanner / Google Flights へのリンクを表示する。

#### フライト詳細（乗り継ぎ便）
- 各区間の詳細：出発地 → 乗り継ぎ空港 → 目的地
- 各区間の便名・機材タイプ
- 乗り継ぎ空港での待ち時間
- 乗り継ぎ空港マップリンク

### 2.3b F-02b: 柔軟な行先・日程検索

#### 概要
「出発地と日程は決まっているが行先は固定しない」スタイルで、複数目的地の最安値を一覧比較する。個人の旅行スタイルに特化した核心機能。

#### 機能詳細
- 出発地・出発日（または期間）を入力し「どこでも検索」を実行
- Amadeus Flight Inspirations API を利用して行先候補を価格順に一覧表示
- 表示内容：目的地名・国旗・最安値・最安値日・フライト時間
- クリックで F-02 の詳細検索へ遷移
- **予算上限フィルター**：「¥30,000以下で行ける場所」絞り込み
- 「国内のみ」「海外のみ」「近距離（4時間以内）」トグル

---

### 2.4 F-03: 価格カレンダー

#### 概要
指定路線の日別・月別最安値をカレンダーで視覚的に表示。

#### 機能詳細
- 月ビュー：各日付に最安価格を表示
- ヒートマップ：価格帯に応じた色分け
  - 最安（緑）→ 平均（黄）→ 高め（赤）
- 往復検索時：出発日 × 帰着日の 2D マトリクス表示（Flexible Dates）
- 「月全体の最安値」バナー表示
- クリックで F-02 検索結果へ遷移

---

### 2.5 F-04: ホテル検索・比較

#### 概要
目的地・宿泊日程・人数を指定してホテルを横断比較表示する。

#### 検索条件
- 目的地（都市・エリア・ホテル名）
- チェックイン日・チェックアウト日
- 宿泊人数・部屋数
- 星評価フィルター
- 施設条件（駐車場・朝食込み・プール等）

#### 表示内容
- ホテル名・写真・星評価
- 1泊あたりの最安値・提供サイト
- 口コミスコア（集約表示）
- 地図上のピン表示
- 「予約サイトへ」ボタン

---

### 2.6 F-06: お気に入り・価格アラート

#### 概要
気になる路線を登録し、指定価格を下回った際に通知する。

#### 機能詳細
- 路線のお気に入り登録（ローカルストレージ or DB）
- 目標価格の設定
- 通知手段：**Web Push 通知**（メール不要、スマホのブラウザ通知）
- 通知頻度：1日1回サマリー
- セールイベント開始時の自動通知
- ※個人利用のためユーザー管理不要。設定はローカルに保存

### 2.7 F-07: 認証（個人利用のため省略）

> 個人のみが利用するため、フルの認証機能は不要。
> バックエンド API には**環境変数に設定した固定 API キー**をヘッダーで送信する形式でアクセス制限する。
> （`X-API-Key: <YOUR_PERSONAL_API_KEY>`）

### 2.8 F-08: AI旅行プランナー（Gemini 活用）

#### 概要
予算・日数・好みを入力するとフライト+観光日程の完全プランを AI が生成する。

#### 機能詳細
- 入力：出発地・予算・日数・旅行スタイル（ビーチ/都市/自然/グルメ）
- Gemini Flash が行先候補・観光スポット・グルメ・日程を生成
- Amadeus API と連携して実際の最安フライトを取得・プランに組み込む
- 生成したプランを PDF またはテキストで保存可能
- 「この行き先のフライトを探す」ボタンで F-02 に遷移

## 3. 非機能要件

### 3.1 パフォーマンス
| 項目 | 目標値 |
|------|--------|
| フライト検索レスポンス | 5秒以内（外部 API 依存あり） |
| カレンダー初期表示 | 2秒以内 |
| 同時接続ユーザー数 | **1（個人利用）** |

### 3.2 可用性
- 個人利用のため稼働率の厳密な要件は設けない
- 外部 API 障害時はキャッシュデータを表示（最終更新時刻を明示）
- Cloud Run のコールドスタートは許容する（無料枠優先）

### 3.3 セキュリティ
- HTTPS 必須
- APIキー認証（`X-API-Key` ヘッダー）による固定アクセス制限（個人利用のため JWT 不要）
- 外部 API キーのサーバーサイド管理（クライアントに非公開）
- OWASP Top 10 対策

### 3.4 対応デバイス・ブラウザ
- **スマートフォン優先**（iPhone Safari / Android Chrome）
- **PWA 対応**（ホーム画面追加・オフラインキャッシュ）
- PC は補助的に対応（レスポンシブ）

### 3.5 多言語・通貨
- 日本語・英語対応（MVP は日本語のみ）
- 通貨：JPY 表示（将来的に多通貨対応）

## 7. システム構成（案）

```
[ユーザー（スマホ / PWA）]
    ↓ HTTPS
[フロントエンド: Next.js / Vercel]
    ↓ REST API (X-API-Key 認証)
[バックエンド: FastAPI / Google Cloud Run]
    ├── フライト検索サービス → Amadeus API（メイン）/ Kiwi API（フォールバック）
    ├── 行先インスピレーション → Amadeus Flight Inspirations API
    ├── ホテル検索サービス   → Amadeus Hotel API
    ├── セール収集サービス   → JAL/ANA/Trip.com RSS + APScheduler（定期ジョブ）
    ├── AI サービス          → Google Gemini 1.5 Flash API
    ├── 通知サービス         → Web Push (pywebpush)
    └── キャッシュ           → DB キャッシュテーブル（Redisなし・個人利用なので十分）
[データベース: PostgreSQL / Neon（既存プロジェクトに travelapp DB 追加）]
[スケジューラ: APScheduler（Cloud Run 内蔵）+ Cloud Scheduler（コールドスタート対策・無料 3ジョブ）]
```

> **Celery + Redis を不採用にした理由**  
> 個人利用のためリクエスト数が少なく、APScheduler でバックグラウンドジョブを処理で十分。  
> Redis ホスティングのコスト・管理コストを削減できる。  
> **Cloud Run コールドスタート対策**: Cloud Scheduler（無料枠 3ジョブ）から定期 HTTP リクエストを送信してコンテナを起動。  
> APScheduler はリクエスト受信後にジョブを実行する構成とする（RSS収集: 毎時、Geminiグラウンディング: 7時/12時/21時、クリーンアップ: 毎日0時）。

## 9. 制約・前提条件

- **個人利用専用**：マルチユーザー・商用公開は想定しない
- **完全無料構成**：有料サービスは一切使用しない（Amadeus / Gemini / Cloud Run / Neon / Vercel はすべて無料枠内）
- 外部 API はメタサーチとして利用（直接予約機能は持たない）
- Trip.com は Affiliate API 経由で正規利用（スクレイピング不可）
- JAL / ANA / LCC セール情報は RSS フィード経由で取得
- Amadeus API の本番環境利用には申請・審査が必要
- フライト検索結果は最大30分 DB キャッシュ（価格変動が激しいため）
- Fly.io はすでに BizPilot で使用中のため本プロジェクトでは使用しない。**Google Cloud Run** を採用
- DB は既存 Neon プロジェクトに **travelapp** データベースを追加作成
